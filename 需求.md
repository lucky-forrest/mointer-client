# 配方管理客户端完整需求文档

## 项目概述
开发一个基于 Python 的配方管理客户端应用程序，包含 GUI 界面和 Socket 通信功能，用于演示配方数据的保存、加载和重置操作。

## 技术栈
- 语言：Python 3.x
- GUI 框架：tkinter
- 通信协议：Socket (TCP)
- 数据格式：JSON
- 线程处理：threading

## 系统架构

### 模块说明
1. **main.py** - 主应用模块
   - 配方管理 GUI 界面
   - Socket 服务端集成
   - 数据收集和展示

2. **socket_server.py** - Socket 服务端模块 (v2)
   - 标准 API 接口实现
   - 支持 save_data、load_data、reset_data 消息类型
   - 纯 JSON 通信格式

3. **socket_client.py** - Socket 客户端测试模块
   - 用于测试服务端功能
   - 包含完整的测试用例

### 区域详解

#### 1. 菜单栏
- **文件**：保存、加载、退出
- **编辑**：重置、全选、启动/停止 Socket 服务端
- **帮助**：关于信息

#### 2. 产品信息区域
- **产品名称**：Entry 控件，用于输入产品名称
- **产品类型**：Combobox 控件，下拉选择（液体/固体/气体/粉末）
- **产品描述**：Text 控件支持多行文本输入，带滚动条

#### 3. 配方信息区域
- **配方名称**：Entry 控件，用于输入配方名称
- **配方编号**：Entry 控件，用于输入配方编号
- **原料选择**：Checkbutton 多选框，可选原料 A-E
- **添加剂**：Combobox 控件，下拉选择（添加剂X/Y/Z）
- **温度设置**：Scale 滑块控件，范围 0-100°C
- **时间设置**：Spinbox 旋转框控件，范围 1-999 秒
- **备注**：Text 控件支持多行文本输入，带滚动条

#### 4. 操作区域
- **保存按钮**：格式化 JSON 数据并弹出显示对话框
- **重置按钮**：清空所有输入控件
- **加载按钮**：加载示例数据到界面
- **退出按钮**：确认后退出程序

#### 5. 状态显示
- **状态标签**：显示操作结果和当前状态
- **进度条**：可视化进度显示（保存/加载操作）

## 交互功能

### 1. 保存功能
1. 点击保存按钮
2. 显示进度条（0% → 100%）
3. 收集表单数据
4. 格式化为 JSON 对象
5. 弹出对话框显示完整 JSON 数据
6. 更新状态标签为"保存成功"

### 2. 加载功能
1. 点击加载按钮
2. 显示进度条（0% → 100%）
3. 模拟从数据库加载数据
4. 将示例数据填入各控件
5. 显示加载成功提示框
6. 更新状态标签为"加载成功"

### 3. 重置功能
1. 点击重置按钮
2. 清空所有输入控件
3. 重置默认值（温度=20°C，时间=30秒）
4. 清空所有复选框选择
5. 更新状态标签为"已重置所有输入"

### 4. 退出功能
1. 点击退出按钮
2. 弹出确认对话框
3. 确认后关闭程序

### 5. Socket 服务端控制
1. 通过菜单：编辑 → Socket 服务端 → 启动/停止
2. 显示服务器运行状态和监听端口
3. 客户端 test_connection() 和 test_load_data()、test_reset_data()

## 数据格式

### 产品信息数据结构
```json
{
  "name": "产品名称",
  "type": "液体/固体/气体/粉末",
  "description": "产品描述文本"
}
```

### 配方信息数据结构
```json
{
  "name": "配方名称",
  "number": "配方编号",
  "ingredients": ["原料A", "原料B", ...],
  "additive": "添加剂名称",
  "temperature": 温度值(0-100),
  "time": 时间值(1-999),
  "notes": "备注文本"
}
```

### 完整数据结构
```json
{
  "产品信息": { ... },
  "配方信息": { ... }
}
```

## Socket 通信协议

### 消息格式
```json
{
  "Request": {
    "Header": {
      "MachineName": "客户端名称",
      "TransactionID": "时间戳",
      "MessageName": "消息类型",
      "UserName": "用户名",
      "EventTime": "yyyyMMddHHmmss.fff"
    },
    "Body": {
      // 业务数据
    }
  }
}
```

### 支持的消息类型

#### 1. save_data (保存数据)
**请求体：**
```json
{
  "ProductInformation": { ... },
  "RecipeInformation": { ... }
}
```

**响应体：**
```json
{
  "DataReturn": {
    "ReturnCode": "OK/NG",
    "ReturnMessage": "返回消息",
    "Data": {
      "save_time": "构建时间",
      "record_id": "记录ID",
      "product_name": "产品名称",
      "recipe_name": "配方名称"
    }
  }
}
```

#### 2. load_data (加载数据)
**请求体：**
```json
{
  "load_condition": "condition"
}
```

**响应体：**
```json
{
  "DataReturn": {
    "ReturnCode": "OK/NG",
    "ReturnMessage": "返回消息",
    "Data": {
      "loaded": true,
      "record_id": null,
      "Data": {
        "产品信息": { ... },
        "配方信息": { ... }
      }
    }
  }
}
```

#### 3. reset_data (重置数据)
**请求体：**
```json
{
  "user_confirm": true
}
```

**响应体：**
```json
{
  "DataReturn": {
    "ReturnCode": "OK/NG",
    "ReturnMessage": "返回消息",
    "Data": {
      "reset_time": "重置时间"
    }
  }
}
```

### 通信配置

- **端口**：8888
- **编码**：UTF-8
- **超时**：10 秒
